# Security Policy

## Reporting a vulnerability

Please do not open a public GitHub issue for security-sensitive reports.

Use one of these private channels instead:
- GitHub Private Vulnerability Reporting for the repository, if it is enabled
- a private GitHub Security Advisory draft, if available
- a direct private contact channel with the maintainer

## What to include

Please include:
- a short summary
- affected versions or commit range
- exact reproduction steps
- impact assessment
- proof of concept only if necessary
- any suggested remediation

## Response expectations

Reports will be triaged privately first.

Please allow time for verification and a fix before public disclosure.
